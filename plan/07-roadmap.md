# 07. ロードマップ

各フェーズは「動くものが確認できる」単位で区切る。フェーズ末に動作確認項目を設ける。

## フェーズ1: 土台 — キャラがデスクトップに立つ

- electron-vite + TypeScript プロジェクト雛形(main / preload / renderer構成)
- 透過・フレームレス・最前面のマスコットウィンドウ
- pixi.js + pixi-live2d-display でサンプルモデル表示(アイドルモーション・目パチ・視線追従)
- ドラッグ移動、クリック透過(キャラ外は下のアプリへ)
- トレイアイコン(表示/非表示・終了)

**確認**: キャラがデスクトップに浮かび、ドラッグでき、キャラ以外の場所は普通にクリックできる

## フェーズ2: 会話 — テキストでLLMと話せる

- LLMProvider 抽象化 + OllamaProvider + OpenAICompatProvider
- 会話オーケストレータ(履歴管理・ストリーミング・文分割・中断)
- バルーンUI(入力欄・発話表示・思考中表示・エラー表示)
- 設定の永続化(electron-store)基盤

**確認**: バルーンから入力するとOllamaの応答が文単位でストリーミング表示される

## フェーズ3: 音声 — キャラがしゃべる

- VoiceEngine 抽象化 + VOICEVOX互換アダプタ
- 文単位TTS + 先読み合成 + 再生キュー
- Web Audio再生 + RMSリップシンク(ParamMouthOpenY)
- TTS障害時のテキストのみフォールバック

**確認**: 応答が音声で再生され、口が動く。VOICEVOXを落としてもテキスト会話は継続する

## フェーズ4: 管理画面

- Reactベースの設定ウィンドウ(LLM / 音声 / キャラ / 挙動 の4タブ)
- モデル一覧・話者一覧の動的取得、接続テスト、テスト再生
- Live2Dモデル差し替え、人格プロンプト編集
- 設定の即時反映

**確認**: 管理画面からLLMモデル・話者を切り替えると即座に会話へ反映される

## フェーズ5: 伺からしさ

- 感情タグ→表情/モーション連動(emotionMap)
- ランダムトーク(アイドル時の自発発話、時刻への言及)
- クリック反応(頭/体タップ→モーション+一言)
- バルーンの自動フェードや細かな所作の調整

**確認**: 放置しているとキャラが勝手にしゃべり、撫でると反応する

## フェーズ6: 配布と仕上げ

- electron-builder で Windows(NSIS)/ macOS(dmg)/ Linux(AppImage)パッケージング
- Live2D Cubism Core同梱のライセンス確認・表記整備
- 初回起動ガイド(Ollama / VOICEVOXのセットアップ誘導)
- README(日本語)・スクリーンショット
- (検討)会話履歴の永続化、複数キャラプリセット

## リスクと対応

| リスク | 対応 |
|---|---|
| Linuxの透過ウィンドウが環境依存 | 既知の制約として明記。Win/macOS優先 |
| ローカルLLMが感情タグ指示に従わない | タグはオプショナル設計(neutralフォールバック) |
| クリック透過のヒットテストが重い | mousemoveスロットリング+ヒットエリア近似から始める |
| pixi-live2d-displayのメンテ状況 | pixi.js v7系で固定。問題があればコミュニティフォーク(v8対応版)へ移行 |
| Cubism Coreの配布条件 | フェーズ6で出版許諾契約の要否を確認(開発中は問題なし) |
