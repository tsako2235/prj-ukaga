# 01. プロジェクト概要

## 目的

「伺か」のようにデスクトップ上にキャラクター(Live2Dモデル)を常駐表示し、裏でローカルLLMと対話するデスクトップアプリを作る。

- キャラはデスクトップに透過・最前面で浮かび、ドラッグで移動できる
- 吹き出し(バルーン)でユーザーと会話する
- 応答生成はローカルLLM(Ollama / OpenAI互換API)
- 応答テキストをVOICEVOX等で音声合成し、リップシンクしながら発話する
- 放置時にキャラが自発的にしゃべる(ランダムトーク)
- 管理画面でLLMモデル・音声エンジン/話者・キャラ人格などを設定できる

## 技術選定

| 領域 | 採用技術 | 理由 |
|---|---|---|
| アプリ基盤 | Electron | Windows / macOS / Linux 対応。透過・フレームレス・最前面ウィンドウの実績が最も多い |
| 言語 | TypeScript(全レイヤー統一) | メインプロセス・レンダラ・管理画面すべて同一言語 |
| ビルド | electron-vite + electron-builder | TS/HMR対応の開発体験と3OS向けパッケージング |
| Live2D描画 | Cubism SDK for Web (Cubism Core) + pixi-live2d-display + pixi.js | WebGLベースでOS非依存。moc3(Cubism 3/4/5)モデル対応 |
| 管理画面UI | React + Vite | コンポーネントベースで設定画面を作りやすい。レンダラ内で完結 |
| 設定永続化 | electron-store | JSONファイルベース。スキーマバリデーション付き |
| LLM | Ollama API / OpenAI互換API | 下記参照 |
| 音声合成 | VOICEVOX互換エンジンAPI | 下記参照 |
| テスト | vitest | ユニットテスト(プロバイダ層・パイプライン層中心) |

### なぜElectronか(Tauri/Flutterとの比較)

- Live2D公式SDKで実用に耐えるのは事実上「Web版」であり、WebGLが動くランタイムが必要
- Electronは透過ウィンドウ・クリック透過(`setIgnoreMouseEvents`)・常時最前面のAPIが安定しており、3OSでの挙動差の知見が豊富
- Tauriはコア部分がRustになり「言語統一」の条件から外れる(自作コードはほぼTSで済むが、透過まわりのOS差異対応でRustに触れる可能性が高い)
- FlutterはLive2D公式サポートが弱くWebView経由になりがち

## ライセンス・配布上の注意

- **Live2D Cubism Core**: プロプライエタリ(無償利用可だが再配布条件あり)。アプリに同梱して配布する場合、Live2D社の「出版許諾契約(Publication License)」の確認が必要。小規模/個人開発は無償枠(直近会計年度の売上1,000万円未満)で利用可
- **pixi-live2d-display**: MIT。ただし内部でCubism Coreを参照するため上記条件は残る
- **Live2Dサンプルモデル**(ひより等): 「無償提供マテリアル使用許諾契約」の範囲で利用可。開発・デモ用に使用し、ユーザー自身のモデル(`.model3.json` 一式)を読み込める設計にする
- **VOICEVOX等の合成音声**: 話者ごとに利用規約が異なる(クレジット表記必須など)。アプリはあくまでローカルAPIを呼ぶだけとし、規約遵守はユーザー責任である旨をREADMEに明記

## 前提とする外部ソフトウェア

アプリ本体には同梱せず、ユーザーが別途起動しているものに接続する:

- Ollama(`http://127.0.0.1:11434`)または OpenAI互換サーバー(LM Studio 等)
- VOICEVOX(`http://127.0.0.1:50021`)/ AivisSpeech(`:10101`)/ COEIROINK 等

接続できない場合は管理画面・バルーンで分かりやすくエラー表示する(起動手順への誘導を含む)。
