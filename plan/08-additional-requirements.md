# 08. 追加要件・バックログ

フェーズ1〜の実装中に出た追加要望をここに記録する。  
本編計画(`01`〜`07`)のスコープ外だが、後続フェーズまたは計画改訂時に取り込む候補。

実装担当は、指示がない限り本ファイルの項目を勝手に実装しない。取り込み時は該当フェーズの計画に落とすか、本ファイルのステータスを更新する。

---

## 記録ルール

各項目は次の形式で追記する。

- **日付**: 要望が出た日
- **ステータス**: `未着手` / `計画取り込み済` / `完了` / `見送り`
- **内容**: 何をしたいか
- **現状**: いまの挙動
- **想定実装**: 方針のメモ（詳細は実装時に詰める）

---

## 要件一覧

### B-001: バルーンを明示的に閉じられるようにする

- **日付**: 2026-07-12
- **ステータス**: 未着手
- **内容**: 一定時間待ちの自動フェード以外に、ユーザーがすぐバルーンを閉じられる操作を追加する
- **現状**: 約12秒操作がなければフェードアウトするのみ。閉じるボタンや即非表示の操作はない
- **想定実装**（候補）:
  - バルーンに「閉じる」コントロールを置く
  - または Escape / バルーン外（キャラ外のウィンドウ領域）クリックで即非表示
  - 非表示中は既存どおりクリック透過。再表示はキャラクリック等（現行どおり）
  - 関連: [`03-mascot-live2d.md`](03-mascot-live2d.md) バルーンUI、「一定時間操作がなければフェードアウト」

### B-002: 管理画面にタブごとの明示的な保存ボタンを置く

- **日付**: 2026-07-12
- **ステータス**: 未着手
- **内容**: 各設定タブ（LLM / 音声 / キャラクター / 挙動）で、変更内容をはっきり保存できるボタンがほしい。いまの即時反映だけだと「保存できたか」が分かりにくい
- **現状**: [`06-admin-ui.md`](06-admin-ui.md) どおり「保存ボタン方式ではなく即時反映」。変更のたびに `settings:set` / `persona:set` している
- **想定実装**（候補）:
  - 各タブに「このタブを保存」ボタンを置く（または画面下部の共通保存）
  - 編集中はローカル state に留め、保存押下でメインへ送る
  - 未保存時は離脱警告や「未保存」表示があるとよい
  - 即時反映を残すか、保存ボタン方式に切り替えるかは取り込み時に決める（現行計画は即時反映）
  - 関連: [`06-admin-ui.md`](06-admin-ui.md) 設定反映の仕組み

### B-003: デバッグモード（ON時はバルーンに感情タグを表示）

- **日付**: 2026-07-12
- **ステータス**: 未着手
- **内容**: デバッグモードを用意し、ON のときは吹き出しに `[happy]` などの感情タグも見えるようにする（動確・LLMのタグ付け確認用）
- **現状**: `emotionParser` が文頭タグを抽出して本文から除去するため、通常時のバルーンにはタグが出ない（モーション連動だけ確認できる）
- **想定実装**（候補）:
  - 設定に `debug.enabled`（または挙動タブの「デバッグモード」トグル）を追加
  - ON 時はバルーン表示用テキストにタグを残す／`[happy]本文` のように再付与して表示
  - OFF（デフォルト）は現行どおりタグ非表示
  - 管理画面から切替。即時反映でよい
  - 関連: [`04-llm-integration.md`](04-llm-integration.md) 感情タグ、[`06-admin-ui.md`](06-admin-ui.md)
