# 08. 追加要件・バックログ

フェーズ1〜の実装中に出た追加要望をここに記録する。  
本編計画(`01`〜`07`)のスコープ外だが、後続フェーズまたは計画改訂時に取り込む候補。

実装担当は、指示がない限り本ファイルの項目を勝手に実装しない。取り込み時は該当フェーズの計画に落とすか、本ファイルのステータスを更新する。

---

## 記録ルール

各項目は次の形式で追記する。

- **日付**: 要望が出た日
- **ステータス**: `未着手` / `計画取り込み済` / `完了` / `見送り`
- **内容**: 何をしたいか
- **現状**: いまの挙動
- **想定実装**: 方針のメモ（詳細は実装時に詰める）

## 要件一覧

### B-001: バルーンを明示的に閉じられるようにする

- **日付**: 2026-07-12
- **ステータス**: 完了
- **内容**: 一定時間待ちの自動フェード以外に、ユーザーがすぐバルーンを閉じられる操作を追加する
- **現状**: バルーン右上の「×」で即非表示。入力欄フォーカス中の Escape でも閉じる。外側クリック閉じはクリック透過のため不可
- **想定実装**（候補）:
  - バルーンに「閉じる」コントロールを置く（第一候補）
  - 入力欄フォーカス中の Escape で即非表示（補助）
  - 非表示中は既存どおりクリック透過。再表示はキャラクリック等（現行どおり）
  - 関連: [`03-mascot-live2d.md`](03-mascot-live2d.md) バルーンUI、「一定時間操作がなければフェードアウト」
- **制約（2026-07-12 レビューで確認）**:
  - 「バルーン外（キャラ外のウィンドウ領域）クリックで閉じる」は**不可**。キャラ外の領域は `setIgnoreMouseEvents(true)` でクリックが下のアプリへ素通しされるため、ウィンドウ側はそのクリックを受け取れない（`plan/03` クリック透過設計）
  - Escape はマスコットウィンドウにキーボードフォーカスがあるとき（実質、入力欄フォーカス中）しか届かない。グローバルショートカット化は他アプリと衝突するため採らない

### B-002: 管理画面にタブごとの明示的な保存ボタンを置く

- **日付**: 2026-07-12
- **ステータス**: 完了
- **内容**: 各設定タブ（LLM / 音声 / キャラクター / 挙動）で、変更内容をはっきり保存できるボタンがほしい。いまの即時反映だけだと「保存できたか」が分かりにくい
- **現状**: 即時反映は維持し、設定・人格・モデル変更の成功時に右下へ「保存済み ✓」トーストを表示
- **想定実装**（候補）:
  - 各タブに「このタブを保存」ボタンを置く（または画面下部の共通保存）
  - 編集中はローカル state に留め、保存押下でメインへ送る
  - 未保存時は離脱警告や「未保存」表示があるとよい
  - 即時反映を残すか、保存ボタン方式に切り替えるかは取り込み時に決める（現行計画は即時反映）
  - **軽量代替案**: 要望の本質が「保存できたか分かりにくい」であれば、即時反映のまま保存成功時に「保存済み ✓」トースト/インジケータを出すだけで足りる可能性がある。まずこちらを検討する ← **採用**
  - 関連: [`06-admin-ui.md`](06-admin-ui.md) 設定反映の仕組み
- **取り込み時の論点（2026-07-12 レビューで確認）**:
  - persona.md は外部エディタ編集を `fs.watch` で監視し `persona:changed` で管理画面へ反映する設計（`plan/04`）。保存ボタン方式にすると「未保存のローカル state」と「外部でのファイル変更」が衝突するため、解決ルール（例: 未編集なら外部変更を自動反映、編集中なら「外部で変更されました: 破棄して再読込 / 自分の編集を維持」の選択を出す）を決める必要がある
  - 軽量案採用により上記衝突は回避

### B-003: デバッグモード（ON時はバルーンに感情タグを表示）

- **日付**: 2026-07-12
- **ステータス**: 完了
- **内容**: デバッグモードを用意し、ON のときは吹き出しに `[happy]` などの感情タグも見えるようにする（動確・LLMのタグ付け確認用）
- **現状**: 挙動タブの「デバッグモード」で切替。ON 時は `[emotion]本文` をバルーン表示。OFF（デフォルト）はタグ非表示
- **想定実装**（候補）:
  - 設定に `debug.enabled`（または挙動タブの「デバッグモード」トグル）を追加
  - ON 時はバルーン表示用テキストにタグを残す／`[happy]本文` のように再付与して表示
  - OFF（デフォルト）は現行どおりタグ非表示
  - 管理画面から切替。即時反映でよい
  - 関連: [`04-llm-integration.md`](04-llm-integration.md) 感情タグ、[`06-admin-ui.md`](06-admin-ui.md)

### B-004: 会話履歴の永続化

- **日付**: 2026-07-12
- **ステータス**: 未着手
- **内容**: アプリ再起動後も会話履歴を引き継げるようにする
- **現状**: 履歴はメモリ保持のみ。再起動でリセットされる（[`02-architecture.md`](02-architecture.md)「当面メモリ保持」、[`04-llm-integration.md`](04-llm-integration.md)・[`07-roadmap.md`](07-roadmap.md) フェーズ6の検討事項）
- **想定実装**（候補）:
  - `{userData}` 配下に JSON 等で履歴を保存し、起動時に読み込む
  - LLM 送信は現行どおり直近 `contextLimit` 往復のみ
  - 履歴クリア（管理画面またはトレイ）があるとよい
  - 保存件数の上限やローテーション方針は取り込み時に決める
  - 関連: [`02-architecture.md`](02-architecture.md)、[`04-llm-integration.md`](04-llm-integration.md)

### B-005: バルーンは直近発話のみ／履歴はボタンで見る

- **日付**: 2026-07-12
- **ステータス**: 完了
- **内容**: バルーンに前回までの会話が一覧表示され続けるのが煩わしい。通常は直近の発話だけ見せ、過去ログは「履歴」ボタンを押したときだけ見られるようにする
- **現状**: 通常は直近ターンのみ表示。ツールバーの「履歴」でセッション内ログを展開、「戻す」で直近表示に戻る。フェード／閉じる時は通常表示をクリア（ログはセッション内保持）。LLM 用メモリ履歴とは分離
- **想定実装**（候補）:
  - 通常表示は直近1往復（または直近のアシスタント発話＋入力欄）に絞る
  - バルーン上に「履歴」コントロールを置き、押下で過去ログを展開／別パネル表示
  - 閉じる／フェード時に蓄積表示をクリアするかは取り込み時に決める（LLM 用のメモリ履歴とは分離してよい）
  - B-004（永続化）とは独立。永続化前でもセッション内履歴の見せ方として実装可能
  - 関連: [`03-mascot-live2d.md`](03-mascot-live2d.md) バルーンUI、B-001（閉じる操作）
